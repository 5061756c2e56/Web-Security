# Protocoles réseau essentiels

Comprendre les protocoles réseau est fondamental pour sécuriser les communications et détecter les menaces. Ce guide couvre les protocoles essentiels et leurs aspects de sécurité.

## TCP/IP

TCP/IP est la suite de protocoles qui régit Internet. Elle est organisée en couches selon le modèle OSI.

### TCP (Transmission Control Protocol)

Protocole de transport orienté connexion qui garantit la livraison fiable des données.

**Caractéristiques :**
- **Connexion orientée** : Établit une connexion (handshake à 3 voies) avant l'envoi de données
- **Fiabilité** : Garantit la livraison des paquets dans l'ordre
- **Contrôle de flux** : Gère le débit de transmission pour éviter la saturation
- **Contrôle de congestion** : Adapte le débit selon l'état du réseau

**Handshake TCP (3-way handshake) :**
1. Client → Serveur : SYN
2. Serveur → Client : SYN-ACK
3. Client → Serveur : ACK

### IP (Internet Protocol)

Protocole de la couche réseau qui gère l'adressage et le routage des paquets.

**Caractéristiques :**
- **Adressage** : Identifie chaque machine sur le réseau avec une adresse IP unique
- **Routage** : Achemine les paquets vers leur destination via des routeurs
- **Fragmentation** : Divise les paquets trop grands pour traverser certains réseaux

**IPv4 vs IPv6 :**
- **IPv4** : 32 bits, ~4,3 milliards d'adresses (épuisées)
- **IPv6** : 128 bits, nombre quasi-illimité d'adresses, sécurité intégrée (IPsec)

## Protocoles de sécurité

### HTTPS (HTTP Secure)

HTTPS combine HTTP avec TLS/SSL pour chiffrer les communications entre le client et le serveur.

**Avantages :**
- **Confidentialité** : Les données sont chiffrées, illisibles pour les intercepteurs
- **Intégrité** : Détection des modifications des données en transit
- **Authentification** : Vérification de l'identité du serveur via certificats

**Port par défaut :** 443

### TLS/SSL Handshake

Le handshake TLS établit une connexion sécurisée en plusieurs étapes :

1. **Client Hello** : Le client envoie les versions TLS supportées, cipher suites, et un nonce
2. **Server Hello** : Le serveur choisit la version TLS et cipher suite, envoie son certificat et un nonce
3. **Vérification du certificat** : Le client vérifie la validité du certificat
4. **Échange de clés** : Génération de clés de session symétriques
5. **Chiffrement activé** : Les données sont maintenant chiffrées

**Versions TLS :**
- **TLS 1.0/1.1** : Obsolètes, vulnérables
- **TLS 1.2** : Standard actuel, sécurisé
- **TLS 1.3** : Version moderne, plus rapide et plus sécurisée

### VPN (Virtual Private Network)

Les VPN créent un tunnel chiffré entre deux points, permettant une communication sécurisée sur un réseau non sécurisé.

**Types de VPN :**
- **VPN site-to-site** : Connecte deux réseaux distants
- **VPN client-to-site** : Permet à un client d'accéder à un réseau distant
- **VPN point-to-point** : Connexion directe entre deux points

**Protocoles VPN :**
- **IPsec** : Standard, très sécurisé, complexe à configurer
- **OpenVPN** : Open source, flexible, largement utilisé
- **WireGuard** : Moderne, performant, configuration simple

## Protocoles de routage

### OSPF (Open Shortest Path First)

Protocole de routage dynamique utilisé dans les réseaux internes (IGP). Calcule les chemins les plus courts selon le coût des liens.

**Caractéristiques :**
- Algorithme de type link-state
- Convergence rapide
- Supporte les réseaux de grande taille
- Authentification possible (MD5, SHA)

### BGP (Border Gateway Protocol)

Protocole utilisé pour router le trafic entre différents réseaux autonomes (AS) sur Internet.

**Caractéristiques :**
- Protocole de routage externe (EGP)
- Utilisé par les FAI et grandes entreprises
- Basé sur des politiques de routage
- Vulnérable aux attaques de hijacking

## Sécurité réseau

### Firewall

Un firewall filtre le trafic réseau selon des règles prédéfinies pour protéger un réseau.

**Types de filtrage :**
- **Filtrage par port** : Autorise ou bloque certains ports
- **Filtrage par adresse IP** : Contrôle l'accès par source/destination
- **Inspection d'état** : Analyse le contexte des connexions (stateful)
- **Filtrage applicatif** : Analyse le contenu des paquets (deep packet inspection)

**Types de firewalls :**
- **Firewall réseau** : Filtre au niveau réseau
- **Firewall applicatif** : Filtre au niveau application
- **Firewall personnel** : Protège un seul ordinateur

### IDS/IPS

- **IDS (Intrusion Detection System)** : Détecte les activités suspectes et génère des alertes
- **IPS (Intrusion Prevention System)** : Détecte et bloque activement les menaces

**Méthodes de détection :**
- **Signature-based** : Détecte des patterns connus d'attaques
- **Anomaly-based** : Détecte des comportements anormaux
- **Heuristic-based** : Utilise des règles et heuristiques

## Vulnérabilités courantes

### Heartbleed (CVE-2014-0160)

Vulnérabilité dans OpenSSL permettant de lire la mémoire du serveur. A affecté des millions de serveurs en 2014.

**Protection :** Mettre à jour OpenSSL, révoquer et renouveler les certificats.

### POODLE (CVE-2014-3566)

Vulnérabilité dans SSL 3.0 permettant le déchiffrement de données. Forçait les serveurs à utiliser SSL 3.0.

**Protection :** Désactiver SSL 3.0, utiliser uniquement TLS 1.2+.

### BEAST (CVE-2011-3389)

Vulnérabilité dans TLS 1.0 permettant le déchiffrement de données. Exploite les faiblesses de CBC.

**Protection :** Utiliser TLS 1.2+ avec des modes sécurisés (GCM).

## Configuration sécurisée

### Recommandations TLS

- Désactiver les versions obsolètes (SSL 3.0, TLS 1.0, TLS 1.1)
- Utiliser TLS 1.2 minimum, TLS 1.3 recommandé
- Choisir des cipher suites sécurisés (éviter RC4, DES, MD5)
- Activer HSTS (HTTP Strict Transport Security)
- Utiliser des certificats valides et à jour

### Configuration firewall

- Principe du moindre privilège : bloquer tout par défaut, autoriser explicitement
- Segmenter le réseau en zones de confiance
- Surveiller et logger toutes les connexions
- Mettre à jour régulièrement les règles

### Monitoring réseau

- Analyser le trafic pour détecter des anomalies
- Surveiller les tentatives de connexion échouées
- Détecter les scans de ports et les attaques
- Utiliser des outils comme Wireshark, tcpdump, Suricata

## Bonnes pratiques

1. **Utiliser HTTPS** pour toutes les communications sensibles
2. **Configurer correctement les firewalls** avec des règles strictes
3. **Surveiller le trafic réseau** pour détecter les menaces
4. **Mettre à jour régulièrement** les équipements réseau et logiciels
5. **Segmenter le réseau** pour limiter l'impact d'une compromission
6. **Utiliser des VPN** pour les accès distants
7. **Implémenter IDS/IPS** pour détecter et bloquer les attaques
8. **Chiffrer les communications** même sur réseaux internes
9. **Auditer régulièrement** les configurations de sécurité
10. **Former le personnel** aux bonnes pratiques de sécurité réseau

## Conclusion

Maîtriser les protocoles réseau est essentiel pour comprendre et sécuriser les communications dans un environnement informatique moderne. La sécurité réseau nécessite une approche en couches : chiffrement (TLS), filtrage (firewall), détection (IDS/IPS), et monitoring continu. Les vulnérabilités comme Heartbleed et POODLE rappellent l'importance de maintenir les systèmes à jour et de suivre les bonnes pratiques de configuration.
